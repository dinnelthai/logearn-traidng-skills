# HTML报表胜率统计功能

**添加时间**: 2026-05-10  
**功能**: 在HTML报表中展示第1次、第2次、第3次...交易的胜率统计

---

## ✨ 新增功能

### 📊 交易次数胜率分析

在HTML报表中新增一个独立的统计区域，展示：
- **第1次交易胜率**
- **第2次交易胜率**
- **第3次交易胜率**
- ...以此类推

### 展示内容

每个交易次数卡片包含：
1. **胜率条形图** - 可视化显示胜率百分比
2. **总交易数** - 该次交易的总数
3. **盈利次数** - 盈利的交易数
4. **亏损次数** - 亏损的交易数
5. **平均收益率** - 该次交易的平均收益率

## 💻 实现细节

### 数据统计逻辑

```python
# 1. 遍历所有CA的交易记录
for r in rows:
    trades = json.loads(r['bt_trades_json'])
    
    # 2. 按trade_number分组统计
    for t in trades:
        trade_num = t['trade_number']  # 1, 2, 3...
        
        # 统计总数、盈利数、收益率
        trades_by_number[trade_num]["total"] += 1
        if t['is_win']:
            trades_by_number[trade_num]["wins"] += 1
        trades_by_number[trade_num]["profit_sum"] += t['profit_rate']

# 3. 计算胜率
win_rate = wins / total * 100
avg_profit = profit_sum / total
```

## 🎯 使用场景

### 1. 策略评估
快速判断策略是否适合重复交易：
- 第1次交易胜率 65% ✅ → 策略有效
- 第2次交易胜率 40% ❌ → 避免重复交易

### 2. 风险控制
根据胜率设置交易规则：
```python
if trade_number == 1 and win_rate >= 60%:
    # 允许第1次交易
    execute_trade()
elif trade_number >= 2 and win_rate < 50%:
    # 禁止第2次及以后的交易
    skip_trade()
```

### 3. 参数优化
对比不同参数下的胜率表现：
- 市值阈值 180k: 第1次胜率 65%
- 市值阈值 200k: 第1次胜率 70%
- 市值阈值 150k: 第1次胜率 55%
